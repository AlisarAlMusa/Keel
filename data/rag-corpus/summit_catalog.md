# Summit College — Course Catalog (RAG corpus)

> Rich advising prose. Hard facts (prerequisites, credits, offering term,
> capacity) are authoritative in the DB, not here. "Recommended preparation" is
> advisory, not a hard prerequisite. One course = one chunk (split on `## `).
> Spans three programs — Computer Science, Data Science, and Chemistry (44
> courses). Per-tenant copy: identical content at seed time, independently
> editable. Tenants differ on policies, sections, and students.

## CS101 — Intro to Programming (3 cr)
Description: Foundations of programming in Python — variables, control flow,
functions, and basic data handling, building toward small working programs.
Topics: syntax, loops, conditionals, functions, simple file I/O.
Skills: reading and writing clean Python, debugging, decomposing problems.
Typical work: weekly coding exercises, two small programs.
Recommended preparation: none — designed as a first course.
Career relevance: the entry point to every software and data role.
Workload note: light-to-medium; steady practice matters more than math.

## CS102 — Data Structures (3 cr)
Description: Core data structures and the cost of using them — lists, stacks,
queues, trees, hash maps, and graphs — with an introduction to complexity.
Topics: arrays vs linked structures, trees, hashing, graph basics, Big-O.
Skills: choosing the right structure, reasoning about time/space cost.
Typical work: implementation assignments, a structures mini-project.
Recommended preparation: comfortable Python from CS101.
Career relevance: the backbone of technical interviews and real systems.
Workload note: medium; the first course where design choices start to matter.

## CS201 — Algorithms (3 cr)
Description: Design and analysis of algorithms — sorting, searching, greedy,
divide-and-conquer, dynamic programming, and graph algorithms.
Topics: recurrences, DP, shortest paths, complexity classes.
Skills: algorithmic thinking, proving correctness, analyzing efficiency.
Typical work: problem sets, algorithm implementation tasks.
Recommended preparation: solid data structures; comfort with discrete math.
Career relevance: central to backend, ML, and research roles.
Workload note: medium-to-heavy; conceptually demanding.

## CS202 — Computer Systems (4 cr)
Description: How programs actually run — architecture, memory hierarchy, and the
operating-system interface beneath high-level code.
Topics: machine representation, caches, memory, system calls.
Skills: low-level reasoning, performance awareness.
Typical work: systems labs, a memory/performance assignment.
Recommended preparation: programming maturity from CS102.
Career relevance: systems, performance, security, and embedded work.
Workload note: heavy; the labs take real time.

## CS210 — Discrete Math for CS (3 cr)
Description: The mathematical language of computer science — logic, sets,
combinatorics, and proof techniques.
Topics: propositional logic, sets/relations, counting, induction, graphs.
Skills: rigorous reasoning, writing proofs, formal thinking.
Typical work: weekly proof problem sets.
Recommended preparation: none specific; comfort with abstraction helps.
Career relevance: underpins algorithms, cryptography, and theory.
Workload note: medium; proof-heavy rather than code-heavy.

## CS302 — Operating Systems (4 cr)
